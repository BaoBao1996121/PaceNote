# 🔍 输出质量自检（AI 自动执行）

**每个工作流 Skill 执行完毕后，AI 必须对照以下信号表自检输出质量：**

| 异常信号 | 可能原因 | 建议提示 |
|----------|----------|----------|
| 章节内容被截断或大段省略 | 模型上下文窗口不足 | 建议使用支持更大上下文的模型（200K+ token） |
| KEY_OUTPUT / CODING_IMPL 标记缺失 | 模型未遵循结构化指令 | 建议切换到指令遵循能力更强的模型 |
| 子Agent 返回空或极短内容 | prompt 过长被截断 | 建议精简输入上下文或拆分步骤执行 |
| 重复套话多、未引用实际代码 | 模型推理能力不足 | 建议使用高级推理模型（3x 级别） |
| 跳过必须执行的步骤 | 推理链断裂 | 建议重试当前步骤或换用更高级模型 |
| 搜索结果为空但应有结果 | 搜索词不准确 | 换用更具体的搜索词或手动指定文件路径 |

**提示格式（统一）**：
```
⚠️ **质量自检提示**:
   检测到: [异常信号描述]
   可能原因: [原因]
   建议: [操作建议]
```

> 🔴 自检自动执行，仅在检测到异常时输出提示。正常情况不输出。
