# 🔄 对话生命周期管理（全局强制）

> 长对话是质量下降的首要原因。每个重型任务应在干净的上下文中执行。

## 对话健康度检测

AI 必须追踪当前对话中已执行的**重型 Skill** 数量（`#pbi-reviewer` / `#roundtable-debate` / `#coding-agent`）。

**触发提醒的条件**（满足任一即触发）：
1. 当前对话已执行 **≥1 个重型 Skill** 后，用户又要求执行**另一个重型 Skill**
2. AI 检测到自身输出出现截断、跳步骤、或质量下降信号

**提醒方式**（使用 `ask_questions`）：

```
header: "新对话"
question: "当前对话已执行过 {已执行的Skill名}，上下文已较长。为确保下一个任务的输出质量，建议新开一个对话窗口执行。\n\n新对话中只需输入：\n`继续 PBI-{ID} 的{下一步任务名}`\nAI 会自动从文件恢复之前的工作成果。"
options:
  - label: "我去新开对话"
    description: "推荐。新对话有完整的上下文窗口，产出质量最佳"
    recommended: true
  - label: "继续在当前对话执行"
    description: "可能出现输出截断或关键步骤遗漏"
```

## 跨对话续接机制

所有重型 Skill 完成时，必须将关键产出写入文件以支持新对话续接：

| Skill | 产出文件位置 | 用途 |
|-------|-------------|------|
| `#pbi-reviewer` | `.copilot-temp/pbi-{ID}-review.md` | 需求返讲文档（含待确认问题） |
| `#roundtable-debate` | `.copilot-temp/roundtable-{NNN}/` | 设计文档（6 个文件） |
| `#coding-agent` | 直接修改源码文件 | 无需额外存储 |

### 持久化内容规范（重要信息 vs 冗余信息）

| 必须保留（写入文件） | 禁止带入（不写入文件） |
|------------------------|---------------------------|
| ✅ PBI ID、标题、模块归属 | ❌ 中间搜索过程（grep_search 结果） |
| ✅ 最终产出文档（返讲/设计） | ❌ 失败的工具调用记录 |
| ✅ 待澄清问题清单及状态 | ❌ 重复读取的文件全文 |
| ✅ 关键技术决策（用了哪个方案、为什么） | ❌ 对话中的来回讨论过程 |
| ✅ 涉及的关键文件路径清单 | ❌ 已淘汰的备选方案 |
| ✅ 下一步行动建议 | ❌ 角色模拟/思考过程 |

### 新对话续接规则

用户在新对话中说"继续 PBI-{ID}"或"继续上次的{任务}"时，AI 必须执行以下恢复流程：

```
步骤1：搜索 .copilot-temp/ 目录
  list_dir(".copilot-temp/")
  → 找到 pbi-{ID}-review.md 或 roundtable-{NNN}/

步骤2：读取产出文件（只读必要部分）
  - pbi-reviewer 产出 → 读取完整文件（含 上下文摘要 区段）
  - roundtable 产出 → 只读 06-design-review.md 的摘要部分
    （不要一次性读取全部 6 个文件，按需读取）

步骤3：向用户确认恢复状态
  使用 ask_questions 展示：
  - 找到的产出摘要（PBI ID、任务类型、完成状态）
  - 下一步建议选项

步骤4：执行下一阶段任务
  基于文件中的内容继续，不依赖上一轮对话的记忆
```

> 🔴 **核心原则**：新对话必须能仅通过读取文件就完全恢复上下文，不依赖任何"上一轮对话的记忆"。

## 多轮修正场景处理（防止上下文污染）

> 用户说"这里不对，改一下"→ 修改后"还是不对"→ 再改→ … 多轮修正会在上下文中留下大量错误版本，污染 AI 判断。

**规则**：

1. **文件始终反映最终版本**：任何对已持久化文档的修正，必须立即**覆写**原文件（不是追加）
   ```
   用户："第3点分析不对，应该是XXX"
   AI：修正文档内容 → 立即 create_file(同路径, 修正后的完整内容) 覆写
   ```

2. **修正超过 3 轮时主动建议新开对话**：
   ```
   header: "新对话"
   question: "当前对话已经进行了多轮修正，上下文中包含较多历史错误版本，可能影响后续输出质量。\n\n建议新开对话输入：\n`继续 PBI-{ID}`\n\nAI 会从文件中读取最新的修正版本（干净上下文，不包含历史错误版本）。"
   options:
     - label: "新开对话"
       description: "推荐。从干净的最新文件开始，不受历史错误影响"
       recommended: true
     - label: "继续当前对话"
       description: "可能受到前几轮错误版本的干扰"
   ```

3. **修正时仅输出差异，不重复全文**：避免在上下文中累积多个完整副本
   - 小范围修正 → 只展示修改的部分（并说明修改原因）
   - 大范围修正 → 重新生成完整文档，但建议新开对话

## Skill 完成时的标准结尾

每个重型 Skill 执行完成后，除了原有输出外，**必须附加**：

```markdown
---
💡 **下一步 & 对话建议**:
- 产出已保存到: `{文件路径}`
- 如需进入下一阶段，建议**新开对话**输入:
  `{推荐的下一步命令，如: "用圆桌会议分析 PBI 1101675"}`
- 在新对话中，AI 会自动读取本次产出作为输入
```
