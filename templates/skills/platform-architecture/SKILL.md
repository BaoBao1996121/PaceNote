# 平台架构知识

> ⚠️ **重要**: 此 Skill 帮助 AI 理解你的软件架构层级，避免混淆"平台/框架"和"应用/业务"代码。
>
> 📌 **此文件需要根据你的项目定制**。下方提供了通用模板和 AI 填充指南。

---

## 🏗️ 架构分层

> 📋 **AI 填充 Prompt** — 复制以下内容发送给 Copilot 生成此部分：
>
> ```
> 请根据我的项目架构，生成一份架构分层图。我的项目结构如下：
> [在此粘贴你的项目目录结构或简要描述]
>
> 请按以下格式生成（从上到下为依赖方向）：
> Layer N: 最上层（业务应用层）- 应用名称和仓库
> Layer N-1: 框架/中间件层 - 通用组件和仓库
> Layer 1: 基础设施层 - 底层服务和工具库
> 每层说明：用途、代码仓库名、主要模块。
> ```

```
项目架构（从上到下）：

┌─────────────────────────────────────────────────────────────┐
│ Layer 3: Applications（业务应用层）                           │
│ ├── 应用 A：[你的应用名]                                     │
│ └── 应用 B：[你的应用名]                                     │
│     代码仓库: [你的应用仓库名]                                │
│     特点: 特定业务场景的逻辑                                  │
├─────────────────────────────────────────────────────────────┤
│ Layer 2: Framework/Platform（框架/平台层）⭐                  │
│ ├── 框架能力: [如 IOC容器、配置驱动、插件系统]                 │
│ └── 通用组件: [如 用户管理、权限、日志、缓存]                  │
│     代码仓库: [你的平台仓库名]                                │
│     特点: 所有应用共用的通用能力                               │
├─────────────────────────────────────────────────────────────┤
│ Layer 1: Infrastructure（基础设施层）                         │
│ ├── 数据库、消息队列、存储服务                                │
│ └── 工具库、算法库                                           │
└─────────────────────────────────────────────────────────────┘
```

## 🔍 搜索策略

### ⭐ 本地优先原则

> 在多根工作区场景下，用户可能同时打开应用项目和平台仓库。
> **必须优先在本地工作区搜索，本地没有时才通过 MCP 远程搜索。**

```
搜索优先级:
1️⃣ 本地工作区（包括打开的平台仓库文件夹）
2️⃣ MCP 远程仓库（本地无结果时）
```

## 🚨 常见混淆场景

### 混淆1: "平台模块" vs "应用内功能"

```
用户: "平台的某功能怎么工作的"

❌ 错误: 在当前应用项目中搜索 → 找到的是应用层调用代码，不是平台实现
✅ 正确: 先在本地查找平台仓库，没有则通过 MCP 搜索远程
```

### 混淆2: 未明确层级

```
用户: "某功能怎么实现的"

✅ 正确: 先判断属于哪一层
   1️⃣ 是否是平台通用组件（见上表）→ 是则搜索平台仓库
   2️⃣ 否则搜索当前工作区
   3️⃣ 不确定 → 两边都搜，并说明来源
```
