# 质量标准与运行时原理

> 本文件由 `app-skill-wizard` Skill 按需加载，包含详细的质量评分标准和运行原理。  
> 人类读者可在 `docs/新应用Skill生成与集成指南.md` 中找到简化版。

---

## 质量检查详细标准

### 单个 Skill 评分（4 项，每项 1 分）

| 项目 | 满分条件 | 扣分条件 |
|------|----------|----------|
| **frontmatter** | `name` 与目录名一致 + `description` 含 ≥3 触发关键词（含中英文） | 缺少 name/description 或关键词 <3 |
| **行数** | 应用层 ≤120 / 平台层 ≤150 | 超过对应上限 |
| **场景→入口** | ≥ 2 个场景且每个都有搜索关键词 | <2 场景或无搜索关键词 |
| **上下游依赖** | 标注了影响程度（高/中/低） | 缺少影响程度 |

### 通过标准

- 每个 Skill 得分 ≥ 3/4 → 通过
- 全部通过 → CP1 检查点放行

### 业务规则豁免

业务规则章节允许标记 `🔴 TODO`，不影响评分。此章节需要领域专家后续补充，AI 无法完全推断。

## YAML frontmatter 规范

```yaml
---
name: {prefix}-{module}-expert          # 必须与目录名完全一致
description: |
  {AppName} {模块名}专家 - {职责描述}。  # 第一行：身份+职责
  触发关键词: {中文1}, {中文2}, {English} # 第二行：触发关键词
---
```

**触发关键词选择策略**：
- 包含模块的中文名（如"数据加载"）
- 包含用户可能的口语表达（如"加载数据"、"导入序列"）
- 包含英文专业术语（如"DataLoad"、"Series Import"）
- 至少 3 个，建议 4-5 个

## skill_categories.json 字段规范

```json
{
  "{category_key}": {
    "display_name": "{GUI显示名}",     // 必填 - GUI 勾选框文本
    "prefixes": ["{app}-"],            // 必填 - Skill 目录名前缀匹配
    "description": "{描述}",           // 可选 - 鼠标悬停提示
    "default_selected": false          // 可选 - 默认勾选状态
  }
}
```

**关键约束**：
- `category_key`：小写英文，与前缀保持一致（去掉末尾连字符）
- `prefixes`：数组，通常只有一个元素，末尾必须有连字符 `-`
- `presets`：预设组合，格式 `"显示名": ["category1", "category2"]`

## 运行时原理

### Copilot 如何发现和触发 Skill

```
① VS Code 启动/Reload Window
    ↓
② Copilot 扫描 .github/skills/*/SKILL.md
    ↓
③ 解析 YAML frontmatter: name + description
    ↓
④ 用户输入问题
    ↓
⑤ 匹配 description 中的触发关键词
    ↓
⑥ 加载匹配的 SKILL.md 正文作为 AI 指令
    ↓
⑦ AI 执行指令（搜索代码、生成文件、分析等）
```

> ⚠️ **关键**: Copilot 只扫描 `.github/skills/`（目标项目），不扫描 `templates/skills/`（本工具）。  
> 因此 Skills 必须通过 GUI 复制到目标项目的 `.github/skills/` 才能生效。

### 过滤机制

```
skill_categories.json 定义前缀
        ↓
GUI 显示多选框（Grid 布局，每行 4 个）
        ↓
用户勾选类别
        ↓
filter_skills_by_categories() 过滤
        ↓
匹配前缀的 Skill + 通用 Skill → 复制到目标项目
```

**通用 Skill 判断逻辑**：目录名不以任何已注册前缀开头 → 视为通用 → 始终包含。

### 多 Skill 协作

```
用户问题 → expert-index 路由 → 匹配领域专家
                                   ↓
                            加载专家 SKILL.md
                                   ↓
                        专家指导 AI 搜索和分析代码
```

## 分批研究质量标准

使用 `runSubagent` 进行分批深度研究时，每批研究报告应包含以下内容：

| 调查项 | 必须 | 说明 |
|--------|:----:|------|
| 目录结构 | ✅ | 列出各层代码子目录及头文件数量 |
| 关键类清单 | ✅ | 3-8 个核心类/接口（精确大小写） |
| 命名空间宏 | ✅ | `*_BEGIN_NAMESPACE` 宏定义及展开结果 |
| 导出宏 | ✅ | `*_Export` 宏名称，标注独立/共享 |
| 拼写陷阱 | ✅ | 非标准拼写（如 `initilize`、`updata`）及具体文件名 |
| CommonInterface | 🔵 平台层 | BE/FE 公共接口清单 |
| XML 配置关联 | 🔵 应用层 | [配置文件] 中的操作/组件注册 注册项 |

**批次大小**：每批 ≤6 个模块（基于实战经验，subagent 深入研究 6 个模块时效果最好）。

**研究报告不合格标准**：
- 缺少关键类清单 → 必须补充研究
- 类名大小写不确定 → 必须 `grep_search` 验证
- 模块目录结构不清 → 必须 `list_dir` 确认

## 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| Skill 生成后 Copilot 没调起 | 未 Reload Window / 不在 Agent 模式 | Ctrl+Shift+P → Reload Window，切 Agent |
| description 没有触发 Skill | 关键词不匹配用户输入 | 增加更多同义词到 description |
| name 与目录名不一致 | Copilot 无法关联 | 确保完全一致 |
| GUI 没显示新类别 | JSON 语法错误 | VS Code 打开 JSON 检查红色波浪线 |
| 选了类别但 Skill=0 | 前缀不匹配 | 确认 Skill 目录名以 JSON 配置的前缀开头 |
| 目标项目中创建的 Skill 不在类别系统中 | 仅本地有效 | 需手动同步回 `templates/skills/` |
