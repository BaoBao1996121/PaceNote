# 默认检查清单

> 当用户未配置自定义检查清单时，使用此默认清单
> 
> 自定义检查清单配置: `data/user_preferences.json` → `docs_config.checklist`

---

## 功能完整性检查

- [ ] 需求描述是否清晰完整？
- [ ] 是否有遗漏的功能点？
- [ ] 边界条件是否明确？
- [ ] 异常场景是否考虑？

## 兼容性检查

- [ ] 是否影响现有功能？
- [ ] 是否需要数据迁移？
- [ ] 是否影响其他模块？

## 非功能性检查

- [ ] 性能要求是否明确？
- [ ] 多模态支持是否考虑？(PT/CT/MR/NM)
- [ ] 多语言是否需要支持？
- [ ] 多分辨率是否需要适配？

---

## 用户配置说明

### 配置文件位置

`data/user_preferences.json`（相对于工作区根目录）

### 可用配置项

| 字段 | 用途 | 使用时机 |
|------|------|----------|
| `docs_config.checklist` | 需求检查清单 | 步骤3 检查时读取 |
| `docs_config.modules` | 功能模块清单 | 步骤2 识别模块时参考 |
| `docs_config.dependencies` | 功能依赖关系 | 步骤3 分析影响时参考 |

### 配置示例

```json
{
  "docs_config": {
    "checklist": ".github/knowledge/需求检查清单.md",
    "modules": ".github/knowledge/功能模块清单.md",
    "dependencies": ".github/knowledge/功能依赖关系.md"
  }
}
```

## 模块识别关键词

> 根据需求内容的关键词，识别涉及的功能模块并调用对应专家

### 动态专家发现

使用 `#expert-index` 获取当前可用的所有专家列表

### 常见关键词匹配

| 关键词 | 推荐专家 |
|--------|---------|
| ROI/测量/标注 | ROI专家 |
| VOI/分割/病灶 | 组织管理专家 |
| 布局/Cell | 布局专家 |
| 保存/导出 | 数据保存专家 |
| DICOM/加载 | 数据加载专家 |
| 窗宽窗位/WL/WW | 密度专家 |
| VRT/MPR/3D | VRT专家 |
| 伪彩/调色板 | 调色板专家 |
| SUV/PET | SUV专家 |
| 联动/同步 | 数据联动专家 |

> 更多专家请查阅 `#expert-index`

## 检查清单结果模板

```markdown
### 3.2 检查清单结果

| 检查项 | 结果 | 备注 |
|--------|------|------|
| 需求描述清晰完整 | ✅ | - |
| 功能点无遗漏 | ⚠️ | 需确认边界场景 |
| 边界条件明确 | ❌ | 未定义最大支持数量 |
| 异常场景考虑 | ✅ | 已包含错误处理 |
| 不影响现有功能 | ⚠️ | 需回归测试 XXX 功能 |
| 无需数据迁移 | ✅ | - |
| 性能要求明确 | ❌ | 未定义响应时间要求 |
| 多模态支持 | ✅ | 支持 CT/MR |
```

图例:
- ✅ 满足
- ⚠️ 待确认
- ❌ 不满足/缺失
