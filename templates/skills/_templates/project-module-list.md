# 功能模块清单模板

> **用途**: 列出项目所有功能模块，供 `#pbi-reviewer`（模块识别）、`#app-skill-wizard`（模块发现）、`#bug-diagnosis-expert`（模块定位）等 Skill 消费。
>
> **生成方式**: 由 `#app-skill-wizard` Step 1f 基于代码扫描结果自动生成，**需人工审核确认**。
>
> **配置路径**: GUI 步骤 ⑤ → `docs_config.modules` → 复制到 `.github/copilot-resources/docs/`

---
## 🤖 AI 填充指南

> **一键生成**: 复制以下提示词发送给你的 Copilot，它会自动扫描你的项目生成此文件：

```
请帮我生成项目的功能模块清单。格式参考 `templates/skills/_templates/project-module-list.md` 中的模板。

项目信息：
- 项目名称: [你的项目名]
- 代码路径: [项目根目录]
- 技术栈: [如: Python/Django, TypeScript/React 等]

要求：
1. 扫描项目目录结构，识别所有功能模块
2. 每个模块填写: 中文名、英文标识、一句话职责、触发关键词(3+)、关键文件
3. 按功能域分组
4. 标注哪些模块依赖平台组件
5. 不确定的项用 ⚠️ 标注
6. 输出完整的 Markdown 表格
```

## 填充规则

1. **模块名称**: 使用团队常用的中文称呼（如"病灶分割"而非"Lesion Segmentation"）
2. **英文标识**: 全小写 + 连字符，与 Skill 命名一致（如 `lesion-segmentation`）
3. **触发关键词**: 至少 3 个，中英文混合，覆盖常见同义词（如"分割, 病灶, Lesion, Seg"）
4. **关键类/文件**: 从研究报告的实际扫描结果中提取，2-4 个核心类名
5. **所属层级**: "应用层"或"平台层"，用于区分 Skill 模板和行数限制
6. **🔍 标注不确定项**: 代码扫描推断的内容用 `⚠️` 标记，提示人工重点审核
