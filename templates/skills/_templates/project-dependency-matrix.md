# 功能依赖关系模板

> **用途**: 描述模块间上下游依赖，供 `#pbi-reviewer`（影响范围分析）、`#bug-diagnosis-expert`（Bug 扩散追踪）等 Skill 消费。
>
> **生成方式**: 由 `#app-skill-wizard` Step 1f 基于代码引用关系自动生成，**需人工审核确认**。
>
> **配置路径**: GUI 步骤 ⑤ → `docs_config.dependencies` → 复制到 `.github/copilot-resources/docs/`

---
## 🤖 AI 填充指南

> **一键生成**: 复制以下提示词发送给你的 Copilot，它会自动分析代码依赖生成此文件：

```
请帮我生成项目的功能依赖关系文档。格式参考 `templates/skills/_templates/project-dependency-matrix.md` 中的模板。

项目信息：
- 项目名称: [你的项目名]
- 代码路径: [项目根目录]
- 已有模块清单: [粘贴你的 project-module-list.md 内容，或说"请先扫描代码"]

要求：
1. 分析每个模块的 import/include/using 语句，识别跨模块依赖
2. 生成依赖矩阵（• 强依赖 / ○ 弱依赖）
3. 为每个模块写出下游影响说明
4. 标注 2-5 条高风险依赖链路
5. 不确定的依赖用 ⚠️ 标注
6. 输出完整的 Markdown 文件
```

## 填充规则

1. **依赖方向**: `A → B` 表示 "修改 A 可能影响 B"（A 是上游，B 是下游）
2. **依赖类型**:
   - `●` 强依赖 = 直接函数调用、接口依赖、类继承
   - `○` 弱依赖 = 事件通知、消息总线、配置关联
3. **影响操作**: 具体到 API/操作级别（如"ROI 创建"、"数据加载"）
4. **依赖来源**: 基于代码中的 `import/using/include` 语句和接口调用分析
5. **🔍 标注推断项**: 无法从代码确认的依赖用 `⚠️` 标注
6. **关键依赖链**: 选 2-5 条修改影响面最大的链路，帮助审查者聚焦
