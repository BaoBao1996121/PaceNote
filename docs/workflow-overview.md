## 阶段 1: 需求返讲 (`#pbi-reviewer`)

```
输入: 需求描述/URL/ID
          │
          ▼
┌─────────────────────┐
│ 1. 解析需求输入      │  ← 支持 URL、ID、标题、纯文本
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 2. 搜索历史知识库    │  ← PBI 索引→详情分层搜索
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 3. 匹配域专家       │  ← expert-index 扫描 frontmatter
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 4. 生成检查清单      │  ← 影响分析 + 风险识别
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 5. 输出返讲文档      │  ← 结构化 Markdown
└─────────────────────┘

产出:
  ✅ 需求返讲文档（含需求概述、影响分析、风险、验收标准）
  ✅ 模块匹配清单（哪些域专家参与）
  ✅ 待确认问题清单
```

## 阶段 2: 圆桌会议 (`#roundtable-debate`)

```
输入: 返讲文档 + 需求上下文
          │
          ▼
┌──────────────────────────┐
│ 上下文收集（7 步）         │  ← 读取所有相关文件和知识
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────┐
│ 上下文评分卡              │  ← 确保信息充足再开始
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────┐
│ Phase A：4 角色并行（subAgent × 4）                       │
│                                                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │ 01-架构派 │  │ 02-效率派 │  │ 03-质量派 │  │ 04-成本派 │ │
│  │ 整体设计  │  │ 实现路径  │  │ 质量保障  │  │ 成本估算  │ │
│  │ 组件交互  │  │ 代码复用  │  │ 测试策略  │  │ 任务拆解  │ │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘ │
│       │              │              │              │       │
│       └──────────────┼──────────────┼──────────────┘       │
│                      │ KEY_OUTPUT 标记                      │
│                      ▼                                      │
│              ┌──────────────┐                               │
│              │ Phase B：     │  ←── 依赖 Phase A 全部完成    │
│              │ 05-实施派     │                               │
│              │ (subAgent×1) │                               │
│              │ 实施规划      │                               │
│              │ 编码骨架      │                               │
│              └──────────────┘                               │
└──────────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────┐
│ Phase C：文档组装          │  ← 整合 5 角色输出
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────┐
│ Phase D：交付评分          │  ← 质量评分卡
└──────────────────────────┘

产出（6 份文件）:
  ✅ 01-architecture.md    （架构决策）
  ✅ 02-efficiency.md      （实现路径）
  ✅ 03-quality.md         （质量策略）
  ✅ 04-cost.md            （成本与任务拆解）
  ✅ 05-implementation.md  （实施规划 + 编码骨架）
  ✅ design-review.md      （综合评审文档）
```

## 阶段 3: 编码实现 (`#coding-agent`)

```
输入: 设计文档（含 CODING_TASK_LIST + CODING_IMPL 标记）
          │
          ▼
┌─────────────────────┐
│ 1. 发现设计文档      │  ← 搜索 .copilot-temp/ 或 docs/
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 2. 解析任务列表      │  ← 提取 CODING_TASK_LIST 标记
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 3. 拓扑排序          │  ← 按依赖关系排序任务
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ 4. 展示执行计划      │  ← 用户确认 ✅
└────────┬────────────┘
         │
         ▼
┌──────────────────────────────────────────────┐
│ 5. 逐任务执行（串行）                         │
│                                               │
│  ┌───────────────────────────────────────┐   │
│  │ Task T1 → subAgent 编码               │   │
│  │   读取 CODING_IMPL:T1 实现骨架        │   │
│  │   编码 → 验证 → 报告                  │   │
│  └───────────────────────────────────────┘   │
│                    │                          │
│                    ▼                          │
│  ┌───────────────────────────────────────┐   │
│  │ Task T2 → subAgent 编码               │   │
│  │   （依赖 T1 完成）                     │   │
│  └───────────────────────────────────────┘   │
│                    │                          │
│                    ▼                          │
│         [HUMAN_CHECK 暂停点]                  │
│         用户审查 → 继续/修改 ✅               │
│                    │                          │
│                    ▼                          │
│  ┌───────────────────────────────────────┐   │
│  │ Task T3 → subAgent 编码               │   │
│  └───────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────┐
│ 6. 完成报告          │  ← 变更文件清单 + 测试建议
└─────────────────────┘
```
