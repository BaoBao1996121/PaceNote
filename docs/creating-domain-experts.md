# 域专家创建指南

> 💡 本指南教你如何为自己项目的业务模块创建域专家 Skill。**所有需要填充的地方都附带了 AI Prompt**，直接复制给 Copilot 即可完成。

---

## 什么是域专家

域专家是一个 SKILL.md 文件，封装了某个业务模块的核心知识：

- **职责边界**: 这个模块做什么、不做什么
- **场景入口**: 什么时候该调用这个专家
- **陷阱表**: 踩过的坑和正确做法
- **依赖关系**: 和其他模块怎么交互
- **配置映射**: 关键配置项在哪里

当 AI 处理涉及这个模块的需求/设计/编码时，会自动加载域专家知识，**给出更精准的结果**。

---

## 创建方式

### 方式 1：AI 向导（推荐，5分钟）

在 Copilot Chat 中输入：

```
#app-skill-wizard 帮我为 [模块名] 创建一个域专家

模块名称: [英文名，如 shopping-cart]
模块中文名: [中文名，如 购物车]
模块职责: [一句话描述，如 管理购物车的增删改查和结算流程]
代码路径: [如 src/modules/shopping-cart/]
```

向导会自动：
1. 搜索你的代码库了解模块结构
2. 生成 SKILL.md 初稿
3. 运行 lint 验证
4. 等你确认后写入

### 方式 2：手动填充模板（10分钟）

1. 复制 `templates/skills/_templates/module-expert-skill.md` 到 `.github/skills/{模块名}-expert/SKILL.md`
2. 按模板中的标注填充各段落
3. 运行 `python scripts/lint_skills.py --skill {模块名}-expert` 验证

## 模板结构详解

每个域专家 SKILL.md 包含以下核心段落。下面对每个段落给出**直接可用的 AI Prompt**。

### 1. YAML Frontmatter（必填）

```yaml
---
name: shopping-cart-expert
description: |
  购物车模块域专家。
  触发关键词: 购物车、cart、加购、结算、SKU管理
---
```

> 📋 **AI 填充 Prompt** — 复制以下内容给 Copilot：
> 
> ```
> 帮我为 [模块名] 生成 SKILL.md 的 YAML frontmatter。要求：
> 1. name 格式为 {模块英文名}-expert
> 2. description 第一行是模块一句话定位
> 3. description 包含"触发关键词:"，列出 5-8 个会触发这个专家的词
>    - 包含中文名、英文名、核心功能动词
>    - 关键词不能和已有 Skill 冲突
> 
> 模块信息：
> - 名称: [模块英文名]
> - 中文名: [模块中文名]  
> - 核心功能: [一句话描述]
> ```

---

### 2. 模块概述

说明模块的核心职责和边界。

> 📋 **AI 填充 Prompt**：
> 
> ```
> 阅读以下代码目录和关键文件，帮我写一个模块概述段落（100-150字）：
> 
> 代码路径: [src/modules/xxx/]
> 
> 需要包含：
> 1. 一句话定位（这个模块做什么）
> 2. 核心职责列表（3-5 条）
> 3. 明确的边界声明（不做什么）
> 
> 请先用 list_dir 浏览目录结构，再读取核心入口文件。
> ```

---

### 3. 场景入口表（关键）

定义什么情况下应该调用这个域专家。

> 📋 **AI 填充 Prompt**：
> 
> ```
> 根据以下模块信息，帮我生成场景入口表。格式：
> 
> | 场景 | 典型需求关键词 | 涉及的核心文件 |
> |------|--------------|--------------|
> 
> 要求：
> 1. 至少 5 个场景
> 2. 每个场景给出 2-3 个典型需求关键词
> 3. 每个场景列出会修改的核心文件路径
> 
> 模块代码路径: [src/modules/xxx/]
> 请先搜索代码中的公开接口和入口点。
> ```

---

### 4. 踩坑陷阱表（最有价值）

这是域专家**最核心的价值**——把踩过的坑写下来防止重复踩。

> 📋 **AI 填充 Prompt**：
> 
> ```
> 帮我为 [模块名] 模块整理踩坑陷阱表。格式：
> 
> | # | 陷阱描述 | 错误做法 | 正确做法 | 严重度 |
> |---|---------|---------|---------|--------|
> 
> 信息来源（请选择你有的）：
> 1. 请搜索这个模块的 git log，找出 revert、fix、hotfix 提交
> 2. 请搜索代码中的 TODO、HACK、WORKAROUND 注释
> 3. 请搜索相关的 Bug Issue/PR
> 
> 模块代码路径: [src/modules/xxx/]
> 
> 如果以上搜索结果不够，我来手动补充我知道的坑：
> - [坑1描述]
> - [坑2描述]
> ```

> 💡 **手动补充建议**: 即使 AI 能搜索到一些，最有价值的陷阱往往是口口相传的经验。建议你至少手写 3 条你亲身踩过的坑。

---

### 5. 依赖关系声明

与其他模块的调用关系。

> 📋 **AI 填充 Prompt**：
> 
> ```
> 分析 [模块名] 的依赖关系，生成依赖声明。格式：
> 
> ### 上游依赖（我调用谁）
> | 模块 | 交互方式 | 典型场景 |
> 
> ### 下游依赖（谁调用我）
> | 模块 | 交互方式 | 典型场景 |
> 
> 请搜索代码中的 import/include 语句，以及对外暴露的 API/接口。
> 模块代码路径: [src/modules/xxx/]
> ```

---

### 6. 配置映射

关键配置项的位置和作用。

> 📋 **AI 填充 Prompt**：
> 
> ```
> 搜索 [模块名] 相关的配置文件，生成配置映射表。格式：
> 
> | 配置项 | 文件路径 | 作用 | 默认值 |
> 
> 请搜索以下可能的配置位置：
> - .env / .env.example
> - config/ 目录下的文件
> - package.json / pyproject.toml 中的配置段
> - 代码中的 getConfig/process.env/os.environ 调用
> 
> 模块代码路径: [src/modules/xxx/]
> ```

## 质量验证

创建完成后，务必运行验证：

```bash
python scripts/lint_skills.py --skill {模块名}-expert
```

检查项包括：
- ✅ YAML frontmatter 格式正确
- ✅ name 和 description 字段存在
- ✅ 触发关键词不与现有 Skill 冲突
- ✅ 文件行数在限制范围内

## 进阶：用 `#project-onboarding` 批量生成

如果你的项目有很多模块需要创建域专家：

```
@ProjectSetup 帮我为这个项目创建完整的 Skill 生态

项目信息：
- 名称: [项目名]
- 技术栈: [如 Python/FastAPI/React]
- 代码路径: [项目根目录]
- 模块列表: [模块1, 模块2, 模块3, ...]
```

ProjectSetup Agent 会批量扫描并生成所有模块的域专家。
